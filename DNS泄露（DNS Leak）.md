**DNS泄露**（DNS Leak）是指在使用 VPN、Proxy（代理）或其他加密网络通道时，原本应该通过加密通道传输的 DNS 查询请求，意外通过本地网络服务商（ISP）的传统 DNS 服务器发送。

这会导致攻击者或 ISP 能够直接监视你访问了哪些网站，从而使网络加密通道的隐私保护功能部分失效。

------

🔍 如何检测 DNS 泄露？

1. 开启你的代理软件或 VPN，确保网络处于代理状态。
2. 访问专业的检测网站：
   - DNSLeakTest.com
   - BrowserLeaks DNS Leak Test
3. 运行 **Standard test** 或 **Extended test**。
4. **结果判断**：
   - **安全**：显示的 IP 和国家全部是你代理节点的服务器（如美国、日本）。
   - **泄露**：列表中出现了你真实所在地的运营商 IP（如中国电信、中国联通、香港宽频等）。

------

🛠️ 解决 DNS 泄露的有效方案

\1. 代理软件开启远程 DNS 解析（远端解析）

- **原理**：将所有 DNS 请求封装，直接发送至远端代理服务器，由服务器代为解析。
- **配置方法**：
  - **Clash / Mihomo**：在配置文件中，将 `dns.enhanced-mode` 设置为 `fake-ip`（虚拟 IP 模式）。这种模式下，本地浏览器拿到的都是虚假 IP，绝不向本地发送真实 DNS 请求。
  - **v2rayN / xray**：在设置中开启“启用高级路由选择”，并在 DNS 设置中将首选 DNS 设为远程加密 DNS（如 `8.8.8.8` 或 `1.1.1.1`）。

\2. 使用安全 DNS 加密协议（DoH / DoT）

- **原理**：使用 HTTPS (DoH) 或 TLS (DoT) 对 DNS 请求进行强加密，ISP 即使拦截也无法读取内容。
- **推荐的公用安全 DNS**：
  - **Cloudflare**：`cloudflare-dns.com`
  - **Google**：`dns.google`
- **配置位置**：
  - **浏览器**：在 Edge/Chrome 设置中搜索“安全 DNS”，选择使用自定义提供商并填入上述网址。
  - **操作系统**：Windows 11 及更新版本的网络设置中，已原生支持对 IPv4/IPv6 启用“仅加密 (DNS over HTTPS)”。

\3. 关闭系统多宿主名称解析（Windows 独有优化）

Windows 会并行向所有网卡发送 DNS 请求以追求最快响应，极易导致泄露。

- **关闭方法**：
  1. 按 `Win + R` 输入 `gpedit.msc` 打开组策略。
  2. 依次进入：`计算机配置` -> `管理模板` -> `网络` -> `DNS 客户端`。
  3. 找到 **“关闭多宿主名称解析”**（Turn off smart multi-homed name resolution），将其双击设置为 **已启用**。
